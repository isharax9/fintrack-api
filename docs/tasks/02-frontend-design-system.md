# Task 02: Frontend Design System

Goal: replace the existing UI identity with a premium professional finance system inspired by `web-designs-to-inspire/`.

## Direction

Chosen style: Option C, professional finance workspace with premium consumer polish.

## Scope

- New color tokens.
- Typography scale.
- Spacing and layout rules.
- Buttons, inputs, tabs, segmented controls, selects, dialogs, tables, cards, charts, badges, tooltips, empty states, skeletons, toasts.
- Dark-first theme with possible light mode later.
- Accessibility rules for color contrast, focus, keyboard flow.

## Inspiration Usage

Use:

- `financial_dashboard_overview_2` for dashboard composition.
- `detailed_transaction_ledger_1` for tables and filters.
- `quick_add_transaction_modal_2` for transaction modal flow.
- `category_budget_planner_1` for budget cards.
- `manage_recurring_transactions` for recurring page.
- `savings_goals_tracker_2` for savings goals.

Do not copy the HTML directly. Build reusable React components.

## Acceptance Criteria

- New token system exists in CSS.
- Existing visual language is removed.
- Components have consistent radius, spacing, focus, hover, disabled, loading, and error states.
- No hardcoded demo finance values in shared components.
- Component primitives are documented enough for app pages.

## Verification

- Run frontend build and lint.
- Visual check desktop and mobile.
- Keyboard navigation check for dialogs and forms.
